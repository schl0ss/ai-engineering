# Context

This repository organizes public AI engineering proof-of-work.

## Language

- **Skill**: a reusable agent workflow stored as Markdown, usually in `skills/<category>/<name>/SKILL.md`.
- **Public project**: an interview-safe artifact that demonstrates engineering shape without exposing private machinery.
- **Private project**: local-only project context that should not be published.
- **Durable source**: state that survives outside a chat session, such as git history, PRs, CI, deploy records, artifacts, or live endpoints.
- **Portfolio proof**: an artifact that a hiring reviewer can inspect to understand engineering judgment.

## Current Pillars

- Agent workflow design through `skills/engineering/anchor`.
- Agent operating-system architecture through the Claude Code operating framework example.
- Data science depth through the Georgia Tech portfolio.
