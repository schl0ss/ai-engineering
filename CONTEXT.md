# Context

This repository organizes public AI engineering proof-of-work for hiring teams.

The thesis: AI engineering is orchestration plus verification. The portfolio should show that Matt can combine ML/data depth, product engineering, agent workflows, guardrails, evals, and production judgment.

## Language

- **Skill**: a reusable agent workflow stored as Markdown, usually in `skills/<category>/<name>/SKILL.md`.
- **Public project**: an interview-safe artifact that demonstrates engineering shape without exposing private machinery.
- **Private project**: local-only project context that should not be published.
- **Durable source**: state that survives outside a chat session, such as git history, PRs, CI, deploy records, artifacts, or live endpoints.
- **Portfolio proof**: an artifact that a hiring reviewer can inspect to understand engineering judgment.
- **AI systems conductor**: the positioning for Matt's work: one engineer coordinating models, agents, data, product surfaces, verification loops, and human gates.
- **Agent coding hours**: 300+ hours in Claude Code, 100+ hours in Codex, 400+ hours total across AI coding agents.
